Semantics

One of the tools we used to achive accessibility for our pages was to follow the guidelines from websites such as W3schools.com and MDN Web Docs. They specified the importance of semantics and the correct use of html tags. This page on HTML Elements from MDN has a list of different elements and their usecases, which we followed to make sure we had the correct tags in our own pages. For example, on this page, which layout is shared by all our pages except Topical, we have a large image at the top of the page, and then the main content further down. Therefore the html code structuring the elements such as the big image, navigation bar and primary header are put inside a <header> </header> while the elements below are all put in a <main> </main>. Within main our elements are supposed to come in pairs of some text and a image. These pairs are grouped by <section> </section> and further grouped by another section for the text and a <picture> </picture> to contain the image and image text. We belive this is a nice structured way to order the different elements and sections, both visually for the user and semantically for the browser and a potential future developer.

Design choices

When designing these pages we wanted a simplistic and clear design that makes it accessible to all. For example, all text is carefully written so it is both concise and easy to understand. Additionally it should also be easy to read, so we made sure all text stands in contrast to its background while also having a decently large font. As we worked on this project we did not just rely on automated tests like Lighthouse to ensure accessibility, we also took advice from guides like this checklist for universal design from UIO

Scalability

Another feature we had in mind when developing this project was the ability to scale the pages when further developing. In the design we used for the introduction page, extra page and this page we have the content in the main part grouped in blocks of one image and a paragraph. If we or a future developer wanted to add another section or paragraph it would be as simple as adding another block. In the picture of the code for the extra page we see that we have three blocks following this pattern of a paragraph accompanied by a picture.
